What was being used

Breadcrumb used on nhs.uk

Previous research

The migration team did some previous work on breadcrumbs.

User need

  • As a user I NEED to know which section of the website I am in SO THAT I am orientated and am aware of other sections.
  • As a user I NEED to know what other sections exist on the website SO THAT I can continue information seeking if the current page doesn't contain the information that I require.
  • As a user I NEED to be able to quickly navigate to a different page in the section SO THAT I can find the most relevant information for my situation.

Design and research

The current breadcrumb when presented on mobile devices took up a lot of space. Searching online, we found a few examples of a truncated breadcrumb on mobile devices.

Breadcrumb on argos.co.uk

Breadcrumb on screwfix.co.uk

It made sense to try this as it as a common pattern on mobile apps.

First iteration

Hypothesis

If we make the breadcrumb background white
Then users will notice it
Because it looks seperate to the header and page background colour.

If we truncate the breadcrumb on mobile
Then users will understand the interaction
Because it is a common mobile design pattern.

Breadcrumb first iteration on desktop

Breadcrumb first iteration on mobile

Research findings

Leeds, pop-up research and lab testing

  • Users understood where they were and where they had come from.
  • Some users used the breadcrumb to navigate back a level.
  • Most users used search to find more info on back pain.
  • Most users clicked the logo to navigate back to the home page.
  • A small number of users expected to find a home link within the menu (on mobile).

Second iteration

We didn't iterate the breadcrumb. We added a home link to the menu on mobile devices.

Hypothesis

If we add a home link to the mobile menu
Then all users will know how to navigate back to the home page
Because there are multiple ways to do so.

Accessibility

Using aria-label="role" provides a label that describes the type of navigation provided in the nav element.

Adding aria-hidden="true" to the svg icons means that they are hidden to screenreaders.

Usage (rules)

When to use this pattern

A breadcrumb should be used on almost every page that you create on NHS.UK. It helps users navigate the many levels of hierarchy that exist.

How it works

  • If the screen size cannot fit the entire breadcrumb trail, the breadcrumb can wrap onto 2 lines, but individual items within the trail shouldn't be truncated.
  • The current page title/name does not need to be shown in the breadcrumb because in the page template, this information is displayed directly underneath as a H1.
  • On mobile, due to the reduced screen width, the full breadcrumb trail is replaced by a 'Back to [parent]' link.
